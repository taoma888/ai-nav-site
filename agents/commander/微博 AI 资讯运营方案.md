# 微博 AI 资讯自动化发布运营方案

## 📋 方案概述

本方案旨在建立一套高效的微博自动化发布流程，每日更新最前沿 AI 消息，打造专业、权威的 AI 资讯账号。

## 二、微博内容风格规范

### 📝 字数要求

| 内容类型 | 字数范围 | 说明 |
|----------|----------|------|
| 快讯类 | 80-120 字 | 简洁明了，核心信息优先 |
| 解读类 | 140-200 字 | 可分段，加话题标签 |
| 深度类 | 200-280 字 | 配合长图或 thread 形式 |

### 🎨 格式规范

```
【标题】用【】或 | 突出主题
【正文】分段清晰，每段 2-3 行
【标签】#AI 前沿# #大模型# #科技资讯#
【来源】文末标注「来源：XXX」
【互动】可加「你怎么看？」引导评论
```

### 🖼️ 配图需求

| 场景 | 配图类型 | 尺寸建议 | 数量 |
|------|----------|----------|------|
| 产品发布 | 官方宣传图/产品截图 | 1080×1080px | 1-3 张 |
| 研究论文 | 论文图表/架构图 | 1200×675px | 1-2 张 |
| 数据报告 | 信息图/数据可视化 | 1080×1350px | 1 张 |
| 人物动态 | 人物照片 + 引语 | 1080×1080px | 1 张 |
| 对比分析 | 对比表格/图表 | 1200×800px | 1 张 |

### 🎯 文案模板

```
模板 1 - 快讯：
【XXX 发布全新 AI 模型】
核心亮点：①... ②... ③...
影响：...
#AI 前沿# #大模型#
来源：XXX

模板 2 - 解读：
【深度 | XXX 为什么重要】
背景：...
关键突破：...
行业影响：...
你怎么看？
#AI 解读# #科技趋势#
```

## 三、发布时间策略

### 🕐 最佳发布时间

| 时段 | 时间 | 适合内容 | 理由 |
|------|------|----------|------|
| **早高峰** | 7:30-8:30 | 隔夜国际资讯 | 通勤时间，阅读率高 |
| **午间** | 12:00-13:00 | 国内上午资讯 | 午休刷微博高峰 |
| **晚高峰** | 18:00-19:00 | 深度解读/汇总 | 下班路上，停留时间长 |
| **睡前** | 21:00-22:00 | 轻松内容/互动 | 睡前刷手机高峰 |

### 📅 推荐发布计划

```
【工作日】
- 08:00 隔夜国际 AI 快讯（1 条）
- 12:30 国内上午资讯（1 条）
- 18:30 深度解读/产品分析（1 条）

【周末】
- 10:00 本周 AI 大事回顾（1 条）
- 20:00 下周看点预告（1 条）

【日均发布】3-5 条
【发布间隔】≥2 小时，避免刷屏
```

### ⚠️ 注意事项

- 避免 23:00-6:00 发布（打扰用户）
- 重大新闻可即时发布（不受时间限制）
- 测试不同时段数据，优化发布时间

## 四、内容筛选标准

### 🎖️ 新闻分级体系

| 级别 | 标准 | 发布优先级 | 示例 |
|------|------|------------|------|
| **S 级** | 必须发，即时发 | 🔴 最高 | OpenAI 发布新模型、重大政策、头部公司战略调整 |
| **A 级** | 当日必发 | 🟠 高 | 主流大模型更新、重要产品发布、融资超 1 亿美元 |
| **B 级** | 选择性发 | 🟡 中 | 技术突破、行业报告、中型公司动态 |
| **C 级** | 一般不发 | ⚪ 低 | 常规产品迭代、小融资、重复信息 |

### ✅ 筛选维度

```
1️⃣ 影响力维度
   - 是否改变行业格局？
   - 是否影响百万级用户？
   - 是否被主流媒体转载？

2️⃣ 时效性维度
   - 是否独家/首发？
   - 信息是否已广泛传播？
   - 是否有后续跟进价值？

3️⃣ 相关性维度
   - 是否与 AI 核心领域相关？
   - 是否符合账号定位？
   - 粉丝是否关心？

4️⃣ 可信度维度
   - 来源是否权威？
   - 信息是否可交叉验证？
   - 是否存在夸大成分？
```

### 🚫 不发布内容

- ❌ 未经证实的传闻/谣言
- ❌ 纯营销软文
- ❌ 过于技术化、难理解的内容
- ❌ 重复信息（已有多家发布）
- ❌ 敏感政治/争议话题
- ❌ 低质搬运内容

## 五、自动化流程设计

### 🔄 工作流程

```
┌─────────────┐
│  信息收集   │ ← RSS/公众号/网站监控
└──────┬──────┘
       ↓
┌─────────────┐
│  内容筛选   │ ← 按分级标准评估
└──────┬──────┘
       ↓
┌─────────────┐
│  内容编辑   │ ← 按格式规范撰写
└──────┬──────┘
       ↓
┌─────────────┐
│  配图制作   │ ← 使用模板/工具生成
└──────┬──────┘
       ↓
┌─────────────┐
│  定时发布   │ ← 微博定时发布功能
└──────┬──────┘
       ↓
┌─────────────┐
│  数据监控   │ ← 阅读量/互动分析
└─────────────┘
```

### 🛠️ 推荐工具

| 功能 | 工具 | 说明 |
|------|------|------|
| 信息聚合 | Feedly/Inoreader | RSS 订阅管理 |
| 内容编辑 | 语雀/Notion | 内容库管理 |
| 配图制作 | Canva/稿定设计 | 模板化设计 |
| 定时发布 | 微博创作者中心 | 官方定时功能 |
| 数据分析 | 微博数据中心 | 官方数据后台 |
| 自动化 | 集简云/腾讯云 HiFlow | 流程自动化（可选） |

## 六、运营目标与 KPI

### 📊 阶段性目标

| 周期 | 粉丝目标 | 阅读量目标 | 互动率目标 |
|------|----------|------------|------------|
| 1 个月 | 1,000+ | 5,000+/条 | 3%+ |
| 3 个月 | 5,000+ | 20,000+/条 | 5%+ |
| 6 个月 | 20,000+ | 50,000+/条 | 8%+ |
| 1 年 | 100,000+ | 200,000+/条 | 10%+ |

### 📈 核心指标

- **阅读完成率**：衡量内容吸引力
- **转发率**：衡量内容价值
- **评论率**：衡量互动质量
- **粉丝增长率**：衡量账号健康度

## 七、风险管控

### ⚠️ 注意事项

1. **版权风险**
   - 图片使用需确认版权
   - 引用内容标注来源
   - 避免直接搬运全文

2. **内容风险**
   - 不发布未经证实消息
   - 不涉及敏感话题
   - 不夸大/误导用户

3. **账号风险**
   - 避免频繁发布（≤5 条/天）
   - 避免营销内容过多
   - 保持内容质量稳定
